# Fable CoT Leak Fix

Fable 模型偶尔会将第二层思考链（CoT）泄漏到正文中，与实际回复粘连在一起，中间没有空格。

本插件在 `on_llm_response` 钩子中检测这一泄漏边界，将泄漏的 CoT 从正文中分离出来，以合并转发消息的形式单独发送。

## 原理

泄漏的 CoT 与正文的粘连处有一个特征：句号后直接跟大写字母，没有空格（例如 `dissect.Yes`）。正常英文写作中句号后必有空格，因此这个 pattern 可以作为切割点。

插件用正则 `[a-z]\.[A-Z]` 找到第一个匹配位置，将句号及之前的内容作为泄漏的 CoT 发送，句号之后的内容覆盖回响应对象作为正文。

## 注意事项

- 这是一个快速补丁，不是根本修复。根本原因是 AstrBot 的 adapter 层对 Fable 模型的 reasoning boundary 解析不正确。
- 如果正文中不存在 `[a-z]\.[A-Z]` 的 pattern，插件不做任何操作，原样返回。
